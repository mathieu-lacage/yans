Introduction

Yans was written as a prototype to investigate the requirements of a complete network simulator with a large number of very different models. The core of the simulation engine has been designed to make the following features as easy as possible to support:

  • dump of trace files in libpcap format,
  • emulation: the ability to plug the simulator in a real network and exchange packets to and from the real network,
  • integration of real-world userspace socket-based applications: minimal changes to the code of such applications and recompilation should be all that is required to run a socket-based application in the simulator,
  • parallelisation: it should be easy to write a single simulation scenario and run it either on a simple workstation or a large cluster of machines and obtain a linear increase of performance with the size of the cluster,
  • scripting: it should be possible to use a scripting language to run various simulation scenarios without requiring a recompile. tcl should not be used.
  • Graphical User Interface: it should be possible to define and run a simulation scenario with a user interface.

While implementing all these features is known not to be a problem, so far, no single simulator has been able to offer easy-to-use solutions to all these problems together which is why yans was started.
